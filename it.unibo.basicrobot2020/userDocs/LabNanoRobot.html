    The robot adpater

    The actor robotAdapterQaStream.kt provides an 'adapter-utility' that allows application designers to send commands the robot actuators (e.g. to execute robot move-commands) without knowing any detail of their low-level configuration.

    Moreover, the adapter works as a an observable data source with respect to the data generated by the robot senors. For example, robot-sonar data can be made available by this component in different forms; for example:
    1. as a stream of values that can be handled by a proper subscriber
    2. as a generator of qak-events, e.g. sonarRobot : sonar( D )
    Since a sonar usually emits data with a quite high frequency and since these data are sometimes not reliable, a good strategy (that we will adopt in the following) is to avoid the direct generation of qak-events by the robotAdapterQaStream.kt.

    Rather, this component could better work as a data-stream source connected with a filter able to select more reliable data and to generate qak-events for those data only.

    Moreover, we can easily add to this data-pipe another filter able to generate other events, e.g. obstacle : obstacle( D ) when the sonar distance is under a prefixed limit.
